# Mitsubishi PLC Communication V3 用户手册

## 1. 简介

Mitsubishi PLC Communication V3 是一款功能强大的三菱PLC通讯工具，专为工业自动化领域设计。本手册将详细介绍如何使用该软件与三菱PLC进行通讯和数据交互。

## 2. 安装与启动

### 2.1 系统要求
- **操作系统**：Windows 7/8/10/11
- **.NET Framework**：4.7.2 或更高版本
- **硬件要求**：
  - CPU：1GHz 或更高
  - 内存：1GB 或更多
  - 网络：以太网接口

### 2.2 安装步骤
1. 下载安装包：从官方网站或发布页面下载最新版本的安装包
2. 运行安装程序：双击安装包文件，按照安装向导的提示完成安装
3. 启动应用：安装完成后，可以从开始菜单或桌面快捷方式启动应用程序

### 2.3 首次启动
首次启动应用程序时，会显示主界面，默认处于未连接状态。

## 3. 主界面介绍

### 3.1 界面布局
主界面分为以下几个主要区域：

1. **菜单栏**：包含文件、操作、工具和帮助菜单
2. **连接状态区域**：显示当前连接状态，提供连接和断开按钮
3. **连接设置区域**：显示连接名称、监控模式和监控间隔
4. **批量操作区域**：包含读写模式、数据数量、类型、地址和操作类型设置
5. **数据表格**：显示添加的操作行，包含数据类型、地址、操作类型、值和结果
6. **日志面板**：显示操作日志和通讯帧

### 3.2 状态栏
状态栏位于主界面底部，显示当前连接状态和系统信息。

## 4. 连接管理

### 4.1 新建连接
1. 点击主界面上的"连接"按钮，或从菜单栏选择"文件" → "新建连接"
2. 在弹出的连接配置窗口中，填写以下信息：
   - **连接名称**：为连接设置一个易于识别的名称
   - **IP地址**：PLC的IP地址（例如：192.168.1.100）
   - **端口**：通讯端口，默认6000
   - **超时时间**：通讯超时时间，默认3000毫秒
   - **协议类型**：选择MC协议格式1
3. 点击"确定"按钮，应用程序将尝试连接到PLC
4. 连接成功后，状态会显示为"已连接"

### 4.2 断开连接
1. 点击主界面上的"断开"按钮
2. 应用程序将断开与PLC的连接
3. 状态会显示为"未连接"

### 4.3 保存连接配置
1. 连接成功后，从菜单栏选择"文件" → "保存配置"
2. 在弹出的保存对话框中，选择保存位置和文件名
3. 点击"保存"按钮，连接配置将被保存为JSON文件

### 4.4 加载连接配置
1. 从菜单栏选择"文件" → "加载配置"
2. 在弹出的打开对话框中，选择之前保存的配置文件
3. 点击"打开"按钮，配置将被加载

## 5. 批量操作

### 5.1 添加操作行
1. 在批量操作区域，设置以下参数：
   - **连接名称**：选择已配置的连接
   - **读写模式**：选择"离散模式"或"连续模式"
   - **数据数量**：在连续模式下，设置要操作的数据数量
   - **类型**：选择软元件类型（如X、Y、M、D等）
   - **地址**：输入软元件地址（如0、100等）
   - **操作类型**：选择"读取"或"写入"
2. 点击"添加行"按钮，将操作添加到数据表格中

### 5.2 删除操作行
1. 在数据表格中选择要删除的行
2. 点击"删除行"按钮，选中的行将被删除

### 5.3 批量读取
1. 在数据表格中添加读取操作行
2. 点击"批量读取"按钮，应用程序将读取PLC中的数据
3. 读取结果将显示在数据表格的"值"列中，"结果"列将显示操作状态

### 5.4 批量写入
1. 在数据表格中添加写入操作行
2. 在"值"列中输入要写入的数据
3. 点击"批量写入"按钮，应用程序将数据写入PLC
4. 写入结果将显示在数据表格的"结果"列中

### 5.5 清除表格
1. 点击"清除表格"按钮，数据表格中的所有行将被删除

## 6. 监控模式

### 6.1 启用监控模式
1. 勾选"监控模式"复选框
2. 设置"监控间隔"（毫秒），默认为1000毫秒
3. 应用程序将按照设定的间隔自动刷新数据

### 6.2 禁用监控模式
1. 取消勾选"监控模式"复选框
2. 应用程序将停止自动刷新数据

## 7. 工具功能

### 7.1 帧分析器
1. 从菜单栏选择"工具" → "帧分析器"
2. 在帧分析器窗口中，输入通讯帧（十六进制格式）
3. 选择"是发送帧"选项，指定帧的类型
4. 点击"解析"按钮，查看分析结果
5. 结果将显示在三个标签页中：
   - **基本信息**：显示帧的基本结构，包括帧头、网络号、站号、命令码等
   - **数据详情**：显示数据部分的详细信息
   - **软元件列表**：显示解析出的软元件信息，包括软元件代码、名称、地址和长度

### 7.2 错误码查询
1. 从菜单栏选择"工具" → "错误码查询"
2. 在错误码查询窗口中，输入错误码（如0x0100）
3. 点击"查询"按钮，查看错误码的解释和解决方案
4. 结果将显示错误码的描述、原因和解决方案

### 7.3 连接监控
1. 从菜单栏选择"操作" → "连接监控"
2. 在连接监控窗口中，查看当前连接状态和通讯统计信息

## 8. 日志管理

### 8.1 查看日志
主界面右侧的日志面板显示操作日志和通讯帧。日志级别包括：
- **INFO**：一般信息
- **DEBUG**：调试信息，包括通讯帧
- **WARNING**：警告信息
- **ERROR**：错误信息

### 8.2 清除日志
1. 点击日志面板下方的"清除"按钮
2. 日志面板中的内容将被清空

### 8.3 保存日志
1. 点击日志面板下方的"保存"按钮
2. 在弹出的保存对话框中，选择保存位置和文件名
3. 点击"保存"按钮，日志将被保存为文本文件

### 8.4 加载日志
1. 点击日志面板下方的"加载"按钮
2. 在弹出的打开对话框中，选择之前保存的日志文件
3. 点击"打开"按钮，日志将被加载到日志面板中

## 10. 通讯帧格式

### 10.1 MC协议帧结构
MC协议通讯帧由以下部分组成：

| 字段 | 长度（字节） | 描述 |
|------|------------|------|
| 帧头 | 2 | 固定为"50 00" |
| 网络号 | 1 | 网络号，默认为0 |
| 站号 | 1 | 站号，默认为0 |
| 预留 | 2 | 固定为"00 00" |
| 命令码 | 2 | 命令类型，如"04 01"表示批量读取 |
| 数据长度 | 2 | 数据部分的长度 |
| 数据 | 可变 | 命令数据 |

### 10.2 常用命令码
| 命令码 | 命令名称 | 功能 |
|--------|----------|------|
| 04 01 | 批量读取 | 读取多个软元件的值 |
| 04 02 | 批量写入 | 写入多个软元件的值 |
| 04 03 | 随机读取 | 随机读取软元件 |
| 04 04 | 随机写入 | 随机写入软元件 |
| 04 05 | 远程运行 | 远程启动PLC |
| 04 06 | 远程停止 | 远程停止PLC |
| 04 07 | 远程复位 | 远程复位PLC |

### 10.3 响应帧结构
响应帧由以下部分组成：

| 字段 | 长度（字节） | 描述 |
|------|------------|------|
| 帧头 | 2 | 固定为"D0 00" |
| 网络号 | 1 | 网络号 |
| 站号 | 1 | 站号 |
| 预留 | 2 | 固定为"00 00" |
| 命令码 | 2 | 与请求帧相同 |
| 数据长度 | 2 | 数据部分的长度 |
| 响应码 | 2 | "00 00"表示成功，其他值表示错误 |
| 数据 | 可变 | 响应数据 |

## 11. 故障排除

### 11.1 连接问题
| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 连接超时 | 网络连接问题 | 检查网络连接，确保电脑与PLC在同一网络 |
| 连接被拒绝 | IP地址或端口错误 | 确认PLC的IP地址和端口号正确 |
| 连接失败 | PLC未运行 | 确保PLC处于RUN状态 |
| 连接失败 | 防火墙阻止 | 检查防火墙设置，确保端口6000允许通过 |

### 11.2 读写问题
| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 读写失败 | 软元件地址不存在 | 确认软元件地址在PLC的有效范围内 |
| 读写失败 | 软元件类型错误 | 确认选择了正确的软元件类型 |
| 读写失败 | 权限不足 | 确保PLC允许远程访问 |
| 读写失败 | 数据类型不匹配 | 确保写入的数据类型与软元件类型匹配 |

### 11.3 应用程序问题
| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 应用程序崩溃 | .NET Framework版本过低 | 安装.NET Framework 4.7.2或更高版本 |
| 应用程序无响应 | 操作过于频繁 | 减少操作频率，增加监控间隔 |
| 内存占用过高 | 日志过多 | 定期清除日志，或调整日志级别 |

## 12. 常见问题解答

### 12.1 如何修改默认端口？
默认端口可以在App.config文件中修改，将"DefaultPort"的值从6000改为其他值。

### 12.2 如何查看通讯帧？
在主界面的日志面板中，选择"DEBUG"级别，应用程序会显示发送和接收的通讯帧。

### 12.3 如何批量读取多个不同类型的软元件？
在离散模式下，分别添加不同类型的软元件操作行，然后点击"批量读取"按钮。

### 12.4 如何设置监控间隔？
在主界面的连接设置区域，修改"监控间隔"文本框中的值，单位为毫秒。

### 12.5 如何保存连接配置？
连接成功后，从菜单栏选择"文件" → "保存配置"，将连接配置保存为JSON文件。
