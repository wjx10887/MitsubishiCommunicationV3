# Mitsubishi PLC Communication V3

## 项目概述

Mitsubishi PLC Communication V3 是一个专为三菱PLC设计的通讯应用程序，提供了直观的用户界面和强大的功能，用于与三菱PLC进行数据交互。

## 功能特性

### 核心功能
- **PLC连接管理**：支持以太网连接，可配置IP地址、端口等参数
- **批量读写操作**：支持同时读写多个软元件
- **实时监控**：可设置监控间隔，实时显示PLC数据变化
- **通讯帧分析**：解析和显示MC协议通讯帧格式
- **错误码查询**：提供常见错误码的解释和解决方案
- **连接监控**：监控PLC连接状态
- **日志记录**：记录操作日志和通讯帧

### 支持的软元件类型
- **位数据**：X（输入继电器）、Y（输出继电器）、M（内部继电器）、L（锁存继电器）、B（位数据寄存器）
- **字数据**：D（数据寄存器）、W（特殊寄存器）、R（文件寄存器）、ZR（字数据寄存器）、T（定时器）、C（计数器）
- **双字数据**：D32（双字数据寄存器）、Float（单精度浮点数）、F64（双精度浮点数）

## 系统要求

- **操作系统**：Windows 7 及以上
- **.NET Framework**：4.7.2 及以上
- **CPU**：1GHz 或更高
- **内存**：1GB 或更多
- **网络**：以太网连接（用于PLC通讯）

## 使用指南

### 连接PLC
1. 点击主界面上的"连接"按钮
2. 在弹出的连接配置窗口中，填写以下信息：
   - **连接名称**：任意名称
   - **IP地址**：PLC的IP地址
   - **端口**：默认6000（可根据实际情况修改）
   - **超时时间**：默认3000毫秒
3. 点击"确定"按钮，应用程序将尝试连接到PLC
4. 连接成功后，状态会显示为"已连接"

### 批量读写操作
1. 在主界面的"批量操作"区域，设置以下参数：
   - **连接名称**：选择已配置的连接
   - **读写模式**：选择"离散模式"或"连续模式"
   - **数据数量**：在连续模式下，设置要操作的数据数量
   - **类型**：选择软元件类型
   - **地址**：输入软元件地址
   - **操作类型**：选择"读取"或"写入"
2. 点击"添加行"按钮，将操作添加到表格中
3. 重复步骤1-2，添加多个操作
4. 点击"批量读取"或"批量写入"按钮，执行操作
5. 操作结果将显示在表格的"结果"列中

### 实时监控
1. 勾选"监控模式"复选框
2. 设置"监控间隔"（毫秒）
3. 应用程序将按照设定的间隔自动刷新数据

### 帧分析器
1. 从菜单栏选择"工具" → "帧分析器"
2. 在帧分析器窗口中，输入通讯帧（十六进制格式）
3. 选择"是发送帧"选项
4. 点击"解析"按钮，查看分析结果
5. 结果将显示在三个标签页中：
   - **基本信息**：帧的基本结构
   - **数据详情**：数据部分
   - **软元件列表**：解析出的软元件信息

### 错误码查询
1. 从菜单栏选择"工具" → "错误码查询"
2. 在错误码查询窗口中，输入错误码
3. 点击"查询"按钮，查看错误码的解释和解决方案

## 技术架构

### 项目结构
- **MelsecPLCCommunicator.Domain**：领域层，包含数据模型和枚举
- **MelsecPLCCommunicator.Application**：应用层，包含业务逻辑和服务接口
- **MelsecPLCCommunicator.Infrastructure**：基础设施层，包含通讯适配器和帧解析服务
- **MelsecPLCCommunicator.UI**：表示层，包含用户界面和交互逻辑

### 核心组件
- **IPlcConnectionService**：PLC连接服务，负责建立和管理与PLC的连接
- **IPlcReadWriteService**：PLC读写服务，负责读写PLC数据
- **ILogService**：日志服务，负责记录操作日志
- **ISettingsService**：设置服务，负责保存和加载配置
- **FrameParserService**：帧解析服务，负责解析通讯帧

### 通讯协议
- 使用三菱MC协议进行通讯
- 支持以太网连接
- 默认端口：6000

## 故障排除

### 连接失败
- **检查网络连接**：确保电脑与PLC在同一网络中
- **检查IP地址**：确认PLC的IP地址正确
- **检查端口**：确认端口号正确（默认6000）
- **检查PLC状态**：确保PLC处于RUN状态

### 读写失败
- **检查软元件地址**：确认地址格式正确
- **检查软元件类型**：确认选择了正确的软元件类型
- **检查PLC权限**：确保PLC允许远程访问

### 应用程序异常
- **查看日志**：在主界面的日志面板中查看详细错误信息
- **检查.NET Framework**：确保安装了.NET Framework 4.7.2或更高版本
