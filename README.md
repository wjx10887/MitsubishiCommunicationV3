# 三菱PLC通信V3

## 项目概述

三菱PLC通信V3是一个专为三菱PLC设计的通讯应用程序，提供了直观的用户界面和强大的功能，用于与三菱PLC进行数据交互。该程序支持多种通信协议，包括MC协议（TCP和UDP）和Modbus TCP协议，适用于各种工业自动化场景。

## 功能特性

### 核心功能
- **多协议支持**：MC协议（TCP/IP、UDP）、Modbus TCP协议
- **PLC连接管理**：支持以太网连接，可配置IP地址、端口等参数
- **批量读写操作**：支持同时读写多个软元件
- **实时监控**：可设置监控间隔，实时显示PLC数据变化
- **通讯帧分析**：解析和显示MC协议通讯帧格式
- **错误码查询**：提供常见错误码的解释和解决方案
- **连接监控**：监控PLC连接状态
- **日志记录**：记录操作日志和通讯帧

### 支持的软元件类型
- **位数据**：X（输入继电器）、Y（输出继电器）、M（内部继电器）、L（锁存继电器）、B（位数据寄存器）、S（状态继电器）、F（特殊继电器）
- **字数据**：D（数据寄存器）、W（特殊寄存器）、R（文件寄存器）、ZR（字数据寄存器）、T（定时器）、C（计数器）
- **双字数据**：D32（双字数据寄存器）、Float（单精度浮点数）、DFloat（双精度浮点数）
- **定时器/计数器**：TN（定时器当前值）、CN（计数器当前值）

## 使用指南

### 连接PLC
1. 点击主界面上的"连接"按钮
2. 在弹出的连接配置窗口中，填写以下信息：
   - **连接名称**：任意名称
   - **协议类型**：选择通信协议（如MC Protocol (3E) - UDP (二进制)、Modbus TCP等）
   - **IP地址**：PLC的IP地址
   - **端口**：默认6000（MC协议）或502（Modbus TCP）
   - **超时时间**：默认3000毫秒
3. 点击"确定"按钮，应用程序将尝试连接到PLC
4. 连接成功后，状态会显示为"已连接"

### 数据读写操作

#### 连续模式
1. 在"操作模式"中选择"连续模式"
2. 设置"数据数量"（要读取的数据点数量）
3. 点击"添加数据"按钮添加数据点
4. 为每个数据点设置：
   - 数据类型：选择数据类型（如D、M、X等）
   - 地址：输入PLC地址
5. 点击"开始"按钮开始连续读取数据
6. 数据会在表格中实时更新

#### 单次模式
1. 在"操作模式"中选择"单次模式"
2. 点击"添加数据"按钮添加数据点
3. 为每个数据点设置数据类型和地址
4. 点击"读取"按钮执行单次读取
5. 读取结果会显示在表格中

### 批量操作

#### 批量读取
1. 在数据表格中添加多个数据点
2. 点击"批量读取"按钮
3. 程序会依次读取所有数据点并显示结果

#### 批量写入
1. 在数据表格中添加多个数据点并设置要写入的值
2. 点击"批量写入"按钮
3. 程序会依次写入所有数据点

### 监控模式
1. 在连接配置中勾选"监控模式"
2. 设置"监控间隔"（毫秒）
3. 连接成功后，程序会按照设定的间隔自动刷新数据

### 工具功能

#### 帧分析器
1. 从菜单栏选择"工具" → "帧分析器"
2. 在帧分析器窗口中，输入通讯帧（十六进制格式）
3. 选择"是发送帧"选项
4. 点击"解析"按钮，查看分析结果
5. 结果将显示在三个标签页中：
   - **基本信息**：帧的基本结构
   - **数据详情**：数据部分
   - **软元件列表**：解析出的软元件信息

#### 错误码查询
1. 从菜单栏选择"工具" → "错误码查询"
2. 在错误码查询窗口中，输入错误码
3. 点击"查询"按钮，查看错误码的解释和解决方案

#### 设备监控
1. 从菜单栏选择"工具" → "设备监控"
2. 在设备监控窗口中查看设备的详细信息和状态

## 技术架构

### 项目结构
- **MelsecPLCCommunicator.Domain**：领域层，包含数据模型和枚举
- **MelsecPLCCommunicator.Application**：应用层，包含业务逻辑和服务接口
- **MelsecPLCCommunicator.Infrastructure**：基础设施层，包含通讯适配器和帧解析服务
- **MelsecPLCCommunicator.UI**：表示层，包含用户界面和交互逻辑

### 核心组件
- **IPlcConnectionService**：PLC连接服务，负责建立和管理与PLC的连接
- **IPlcReadWriteService**：PLC读写服务，负责读写PLC数据
- **ILogService**：日志服务，负责记录操作日志
- **ISettingsService**：设置服务，负责保存和加载配置
- **FrameParserService**：帧解析服务，负责解析通讯帧
- **ICommunicationAdapter**：通信适配器接口，定义了与PLC通信的方法
- **NetworkCommunicationAdapter**：网络通信适配器，实现了以太网通信
- **SerialCommunicationAdapter**：串口通信适配器，实现了串口通信

### 通讯协议
- **MC协议**：支持TCP/IP和UDP连接
- **Modbus TCP协议**：支持标准Modbus TCP通信
- **默认端口**：MC协议默认6000，Modbus TCP默认502

## 故障排除

### 连接问题

#### 连接失败，提示「无法连接到设备」
- **检查网络连接**：确保电脑与PLC在同一网络中
- **检查IP地址**：确认PLC的IP地址正确
- **检查端口**：确认端口号正确（MC协议默认6000，Modbus TCP默认502）
- **检查PLC状态**：确保PLC处于RUN状态
- **检查防火墙**：确保防火墙允许通信端口

#### UDP连接成功，但读取数据失败
- **检查PLC的UDP设置**：确保PLC的UDP功能已启用
- **增加超时时间**：在连接配置中增加超时时间
- **减少批量读取数据点**：避免一次性读取过多数据点

### 通信问题

#### 批量读取时出现超时错误
- **增加UDP超时时间**：在NetworkCommunicationAdapter.cs中设置更长的超时时间
- **添加读取延迟**：程序已在批量读取之间添加了50ms延迟
- **减少批量读取数量**：单次批量读取的数据点不宜过多

#### Modbus TCP连接失败
- **检查PLC的Modbus TCP设置**：确保PLC的Modbus TCP功能已启用
- **检查端口号**：确认使用正确的端口号（默认为502）
- **检查Modbus地址**：确保使用正确的Modbus地址格式

### 数据问题

#### 读取的数据值不正确
- **检查数据类型**：确认选择了正确的数据类型
- **检查地址格式**：确保地址格式符合所选协议的要求
- **确认PLC数据**：确认PLC中该地址的数据是否存在

#### 写入数据失败
- **检查PLC状态**：确保PLC处于可写入状态
- **检查地址权限**：确认地址是否可写
- **检查数据类型**：确保数据类型和值匹配

### 应用程序异常
- **查看日志**：在`logs\network.log`文件中查看详细错误信息
- **检查.NET Framework**：确保安装了.NET Framework 4.7.2或更高版本
- **重启应用**：尝试关闭并重新启动应用程序
